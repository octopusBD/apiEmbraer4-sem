## Desafio
Atualmente muitos pilotos trabalham como “freelancer” e não operam apenas uma aeronave/frota, o que dificulta o reconhecimento simples e fácil de certos equipamentos e versões de softwares instalados na aeronave. Além disso, nem sempre o piloto tem acesso de forma rápida a todos os manuais e documentos da aeronave, o que prejudica seu conhecimento sobre a mesma e suas capacidades.

<p align="right">(<a href="#top">voltar ao topo</a>)</p>


<li> Requisitos Funcionais</li>
      <ul>
            <li>Utilização de API’s para futura integração com sistemas em plataforma web</li>
            <li>Acesso à base de dados de clientes (usuário, senha e equipamento)</li>
            <li>Usuários devem ter perfis diferentes (administrador, edição e consulta)</li>
            <li>Possibilidade de aplicação de filtros de seleção (por categoria de item)</li>
            <li>Visualização de arquivos PDF no App e possibilidade de exportação no mesmo formato, com registro da data de geração do arquivo e do usuário.</li>
      </ul>
</ul>

<br>

<li> Requisitos Não Funcionais</li>
      <ul>
            <li>O sistema deve ser protegido contra acesso não autorizado, garantindo que apenas usuários autenticados possam realizar operações no sistema.</li>
            <li>O sistema deve ser capaz de processar uma grande quantidade de dados de forma eficiente, sem apresentar atrasos ou lentidão para os usuários.</li>
            <li>O sistema deve estar disponível para uso a maior parte do tempo, minimizando o tempo de inatividade para manutenção ou atualizações.</li>
            <li> O sistema deve ser fácil de usar e intuitivo, permitindo que os usuários realizem as operações de forma rápida e eficiente, sem a necessidade de treinamento adicional.</li>
      </ul>
</ul>

<p align="right">(<a href="#top">voltar ao topo</a>)</p>

## Solução
A equipe da Octopus desenvolverá um software web para a empresa Embraer que possibilitará aos usuários acessar toda a documentação de forma simples e intuitiva, por meio de dispositivos móveis, tornando possível o acesso ao manual em qualquer lugar e a qualquer momento.

## Backlog


<li> Sprint 1:</li>
      <ul>
            <li>Estruturação e persistência de dados ao banco;</li>
            <li>Desenvolver a interface de consulta com a seguinte funcionalidade: (Permitir a visualização da lista de itens opcionais da aeronave)</li>
            <li>Configurar sistema de notificações de atualizações;</li>          
            <li>Filtros.</li>
      </ul>
</ul>

<br>

<li> Sprint 2:</li>
      <ul>
            <li>Criar interface de login e autenticação para cada tipo de usuário;</li>
            <li>Desenvolver a interface parcial do administrador com as seguintes funcionalidades: Criar, modificar e remover clientes da base de dados. </li>
      </ul>
</ul>

<br>

<li> Sprint 3:</li>
      <ul>
            <li>Desenvolver a funcionalidade para gerar visibilidade estatística;</li>
            <li>Desenvolver a interface de edição com a seguinte funcionalidade: Inserir, modificar e remover novos itens da base de dados apresentados aos clientes.</li>
      </ul>
</ul>

<br>

<li> Sprint 4:</li>
      <ul>
            <li>Gerar relatório;</li>
            <li>Testar e validar o sistema;</li>
            <li>Preparar a documentação para os usuários finais.</li>
      </ul>
</ul>
